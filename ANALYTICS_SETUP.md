### How Web Vitals Are Collected

The `analytics-setup.js` uses the **PerformanceObserver API** to:

1. **Monitor layout shifts** (CLS)
   - Detects unexpected visual movements
   - Rates as "good" or "poor"

2. **Track user interactions** (FID/INP)
   - Measures delay between user input and response
   - Tests responsiveness

3. **Watch content loading** (LCP)
   - Tracks when largest element loads
   - Indicates perceived loading speed

4. **Record paint metrics** (FP/FCP)
   - First pixel painted
   - First content pixel painted

All metrics are tagged with "good" or "poor" rating based on Google's Core Web Vitals thresholds.

## 🚀 What to Monitor

### Key Metrics to Watch
- **Visitors**: Track growth over time
- **Page Views**: Measure engagement depth
- **Bounce Rate**: Higher = less engaging (aim < 50%)
- **Avg Session Duration**: Longer = more engaging
- **Device Split**: Optimize for your top devices
- **Geographic Distribution**: Plan localization

### Performance Targets (Core Web Vitals)
| Metric | Good | Needs Work | Poor |
|--------|------|-----------|------|
| **LCP** | < 2.5s | 2.5-4s | > 4s |
| **FID** | < 100ms | 100-300ms | > 300ms |
| **CLS** | < 0.1 | 0.1-0.25 | > 0.25 |
